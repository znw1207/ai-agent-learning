# C 语言基础学习 Roadmap（Foundation）

定位：本主题是支线能力，用于补齐底层理解，不以业务开发为主。

## 1. 服务的主线场景

- `topics/system-design`：理解内存、性能、系统调用对服务行为的影响。
- `topics/agent`：理解运行时开销、序列化和底层效率问题。
- `topics/rag`：理解向量计算/检索库的底层性能约束。

## 2. 学习范围

- 语言基础：指针、数组、结构体、内存管理。
- 系统基础：进程/线程、文件 I/O、网络 I/O 基础。
- 调试能力：gdb、valgrind、编译与链接基础。
- 性能意识：缓存局部性、拷贝成本、常见瓶颈定位。

## 3. 产出要求

- `knowledge/`：底层概念文档（内存、I/O、并发）。
- `examples/`：小型程序（字符串处理、socket 基础、内存示例）。
- `qa/`：底层问题问答（偏工程理解）。

## 4. 第一周任务

- [ ] 写一个动态内存分配与释放示例并做错误检查。
- [ ] 写一个基础文件读写程序并记录性能观察。
- [ ] 总结“C 的底层知识如何帮助 Agent 工程优化”。

## 5. 验收标准

- 能解释常见内存错误及排查方式。
- 能读懂简单 C 项目的构建流程。
- 能把底层知识转化为主线项目优化思路。
