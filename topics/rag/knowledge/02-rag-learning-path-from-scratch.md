# RAG 学习路径（从 0 到 Agentic RAG）

这份路径专门配合 `topics/rag/examples/` 的 4 个脚本使用。

## 1. 学习目标

你要逐步掌握 4 件事：

1. 检索是什么
2. 传统 RAG 的完整链路
3. RAG 的评估关注点
4. Agentic RAG 比传统 RAG多了什么

## 2. 分阶段学习

## Step 1：检索基础（脚本 01）

你要理解：
- 为什么先检索再回答
- 关键词检索的优点和局限

完成标准：
- 能解释为什么关键词检索会漏掉同义词

## Step 2：向量检索直觉（脚本 02）

你要理解：
- TF-IDF 和 cosine 的直觉
- 为什么向量检索比纯关键词更鲁棒

完成标准：
- 能解释 top-k 是怎么得到的

## Step 3：传统 RAG 流程（脚本 03）

你要理解：
- `retrieve -> augment -> answer`
- 上下文构造会直接影响答案质量

完成标准：
- 能画出传统 RAG 的数据流

## Step 4：Agentic RAG（脚本 04）

你要理解：
- 为什么要多轮检索
- query rewrite 的价值
- 什么时候应该停止继续检索

完成标准：
- 能解释 Agentic RAG 和传统 RAG 的本质区别

## 3. 两种 RAG 的最小对比

传统 RAG：
- 固定流程
- 通常检索一次
- 结构简单，成本更稳

Agentic RAG：
- 动态决策流程
- 可多轮检索和 query 改写
- 质量上限更高，但复杂度和成本也更高

## 4. 你下一步应该做什么

完成这 4 个脚本后再做：

1. 把语料换成你自己的学习笔记
2. 增加 10 条问答样本做小评估
3. 比较“单次检索 vs 多轮检索”的效果差异

这三步做完，你就不是只懂概念，而是有基础实践了。
