# AGENTS.md

本文件用于定义 `ai-agent-learning/` 内 AI 协作规则。
关键词采用中文为主，必要时保留英文术语（如 `workflow`、`check`、`scope`）。

## 0) 内容组织优先级

- 本仓库的核心产出是三类：`知识点文档`、`examples`、`问答`。
- 优先顺序：知识点文档 > 可执行示例 > 问答整理。
- 问答用于训练表达，不应替代系统化知识文档。
- 目录组织采用知识点优先（topic-first）：例如 `topics/rag/{knowledge,examples,qa}`。

## 0.1) 主线与支线边界

- 主线（默认优先）：`topics/agent`、`topics/rag`、`topics/system-design`、`topics/interview`。
- 支线（基础能力）：`topics/foundation/*`，例如编程语言、网络、操作系统等。
- AI 默认优先处理主线任务；支线按需补充，服务主线能力提升。
- 新增支线内容时，必须在文档中说明它服务于哪个主线场景。

## 1) 项目使命

- 持续提升对 AI Agent 的理解：理论学习、代码实战、复盘总结三位一体。
- 将 `ai-agent-learning/` 维护为个人长期学习与实验仓库（learning lab）。

## 2) 管理边界（非常重要）

- 本文件只约束 `ai-agent-learning/` 目录内的任务与改动。
- `opencode/`、`langgraph/`、`AgentGuide/`、`component-app-webservice/` 在这里仅作为学习参考（reference），不属于本目录治理范围。
- AI 在 `ai-agent-learning/` 中可以引用外部仓库经验，但不得把“外部仓库规范”当作本目录强制规则。

## 3) AI 默认工作流

收到任务后，默认按以下 `workflow` 执行：

1. 澄清目标、输入输出和约束。
2. 定位涉及文件与依赖关系。
3. 先做最小可行改动（MVP change）。
4. 执行可用验证（脚本、示例运行、基础检查）。
5. 输出影响范围、验证结果和下一步建议。

补充规则（回答策略）：

- 简单问题：可直接基于通用知识回答，并给出清晰结论。
- 复杂问题（例如深入 RAG、生产级架构选型、评测体系）：先补充高质量资料，再输出结构化答案。
- 对复杂问题的输出要求：必须包含“参考来源类型 + 你的工程化总结 + 可执行下一步”。

## 4) 质量门槛（Required）

每次非琐碎改动，AI 必须做到：

- 说明影响范围（impact scope）：改了什么、可能影响哪里。
- 执行可用验证并如实报告结果：跑了哪些命令，哪些未跑。
- 给出 1-3 条下一步建议（next steps）。

建议验证方式：

- 在 `ai-agent-learning/agent-demo/` 下优先运行相关示例脚本。
- 示例：`uv run python 01_state_graph_basics.py`

## 5) 学习导向原则

- 实用优先：小步迭代，快速验证。
- 关键改动留痕：结论要可追溯（命令、输出、文件位置）。
- 文档和代码保持一致，避免“代码变了但笔记没更新”。
- 不编造未验证结论；不虚报测试执行情况。
- 遇到复杂主题时，先补资料再下结论，避免“只靠记忆硬答”。

## 5.1) 复杂主题资料引入规则

- 可引入形式：外部仓库、官方文档、技术博客/论文。
- 引入标准：内容质量高、与当前学习阶段直接相关、可落地到本仓库产出。
- 引入后必须沉淀：至少新增 1 份知识文档或 1 个可执行示例。
- 资料应尽量放在 `external/`（子模块）或在文档中明确来源和用途。

## 6) agentV3 学习策略（仅参考，不代管）

`agentV3` 位于 `component-app-webservice/`，不由本目录直接管理。
在 `ai-agent-learning/` 中，仅可把它作为案例研究对象（case study），重点学习：

- 如何借鉴 `opencode` 的架构思想到真实业务场景。
- 如何设计任务输入输出契约（contract）与运行时上下文。
- 如何补齐稳定性能力：重试、超时、降级、日志与可观测性。

说明：若需要制定 `agentV3` 的执行规范，应在 `component-app-webservice/` 内单独维护对应文档。

## 6.1) 外部仓库子模块策略

- 允许把参考仓库以 `git submodule` 方式挂到 `ai-agent-learning/external/`。
- 这些子模块默认是“学习参考”，不是本仓库直接代管的业务代码。
- 当子模块更新后，需要在本仓库文档中补充“学到了什么、落地了什么”。

## 7) AI Agents 面试必考知识点（学习输出优先覆盖）

- Agent 基础循环：planning、acting、tool use、reflection。
- 工作流范式：ReAct、Plan-and-Execute、多 Agent 协作、Map-Reduce 分发。
- 状态与记忆：短期状态、长期记忆、检索边界。
- 工具与安全：工具协议、权限边界、风险控制。
- 评估与可观测性：成功指标、trace/log 分析、失败分类。
- 框架选型：LangGraph 与其他方案的差异和适用场景。
- 生产要素：延迟、成本、稳定性、回滚策略。

## 8) 实验记录模板

重要实验建议在相关文档追加如下模板：

```md
## [实验标题]
- Date: YYYY-MM-DD
- Goal: 要解决的问题
- Context: 相关模块/路径 + 关键假设
- Approach: 核心方案与理由
- Result: 成功点 / 失败点
- Evidence: 命令、测试、日志片段
- Next: 1-3 条后续行动
```

## 9) AI 输出约定

实现类任务的回复应至少包含：

- 改了什么（文件与意图）。
- 影响范围（谁会受影响）。
- 验证情况（已执行/未执行）。
- 下一步建议（1-3 条）。

当任务是复杂主题时，额外补充：

- 引入了哪些资料（仓库/文档类别）。
- 这些资料如何被转化为本仓库的知识文档或示例。

若本文件与用户的明确指令冲突，以用户指令优先。
